1. 시스템 개요

본 프로그램은 보험 설계사들이 고객 관리, 기존 보장 분석, 가입 제안 이력, 그리고 종이 청구서 및 심사 서류 파일을 **구글 드라이브 클라우드** 또는 **개인 PC 저장공간**을 통해 효과적으로 제어할 수 있는 스마트 상담 차트 웹 솔루션입니다.

설계사 개인의 Google 계정과 직접 연동되므로, 어떠한 외부 서버에도 고객의 정보가 유출되지 않아 **개인정보보호(영업 기밀)** 측면에서 완벽한 독립성과 안정성을 보장합니다.

2. 구글 클라우드 초기 세팅 방법 (최초 1회 필수)

본 프로그램을 모바일이나 외부에서 원활하게 사용하려면 설계사 본인의 구글 개발자 프로젝트(GCP)에서 키를 발급받아 환경 설정에 입력해야 합니다.

[1단계] 구글 API 콘솔 접속 및 프로젝트 생성

  1. 웹 브라우저에서 구글 클라우드 콘솔에 접속하여 로그인합니다.
  2. 좌측 상단의 프로젝트 목록을 클릭하고 [새 프로젝트]를 만듭니다 (예: InsureChart-Client).

[2단계] 필수 API 라이브러리 활성화

사용자 정보, 고객 폴더 검색, 지도 마킹 및 캘린더 등록을 위해 다음 4개의 API를 반드시 활성화해야 합니다.

  1. 콘솔 상단 검색창에 아래 명칭을 각각 검색하여 [사용] 버튼을 클릭합니다.
    • Google Drive API (파일/폴더 저장)
    • Google Calendar API (생일/계약일 일정 연동)
    • Geocoding API (주소 좌표 변환)
    • Maps JavaScript API (고객 위치 지도 렌더링)

[3단계] OAuth 동의 화면 및 범위 지정

  1. 좌측 메뉴에서 [API 및 서비스] -> [OAuth 동의 화면]으로 진입합니다.
  2. User Type을 [외부(External)]로 설정하고 만들기 버튼을 클릭합니다.
  3. 앱 정보 입력 후 [범위 추가 또는 삭제] 버튼을 누르고 아래의 범위를 반드시 추가합니다.
    • .../auth/drive.file (설계사가 등록한 파일 및 폴더 읽기/쓰기 권한)
    • .../auth/calendar.events (구글 캘린더 일정 추가/수정 권한)
  4. 테스트 사용자 탭에서 **[사용자 추가]**를 눌러 로그인할 본인의 구글 이메일을 등록해 줍니다.

[4단계] 인증 정보 발급 및 도메인 등록

  1. 좌측 메뉴 [사용자 인증 정보]로 이동합니다.
  2. [+ 사용자 인증 정보 만들기] -> [API 키]를 눌러 API 키를 발급받습니다.
  3. 다시 [+ 사용자 인증 정보 만들기] -> [OAuth 클라이언트 ID]를 클릭합니다.
    • 애플리케이션 유형: 웹 애플리케이션 선택
    • 승인된 JavaScript 원본에 아래 주소를 입력합니다:
      http://localhost:3000, http://localhost:5000, 그리고 실제 운영 주소인 https://onions-lab.xyz
  4. 생성 버튼을 누르면 나오는 [OAuth 클라이언트 ID]와 [API 키] 값을 복사합니다.

3. 최초 접속 시 프로그램 환경 설정

  1. 배포된 사이트 주소(예: https://onions-lab.xyz)에 접속합니다.
  2. 화면 우측 상단의 **[구글 클라우드]** 모드를 클릭하여 선택합니다.
  3. 그 옆의 ⚙️ (설정) 아이콘을 클릭하여 발급받은 **[OAuth Client ID]**와 **[API Key]**를 차례로 입력하고 **[설정 저장 및 연동]**을 누릅니다.
  4. 우측 상단의 파란색 **[구글 연동 로그인]**을 눌러 본인의 계정으로 구글 로그인을 승인합니다.
📁 폴더 구조 자동 부트스트랩 기능:
로그인이 완료되면 본인의 구글 드라이브 내에 01_장기보험_고객관리라는 루트 폴더와 그 아래 자음별 서브 폴더들(ㄱ, ㄴ~ㅅ, ㅇ, ㅈ~ㅎ 등)이 없을 시, 시스템이 자동으로 감지하여 **해당 표준 디렉터리를 100% 자동으로 구축**해 줍니다.

4. 주요 핵심 기능 사용 가이드

📍 신규 고객 추가 및 폴더 매칭

  • 사이드바 검색칸 우측의 `+` (초록색 버튼)을 누르면 신규 등록 모달이 열립니다.
  • 고객의 이름, 주소, 생년월일, 계약일을 입력한 뒤 완료를 누르면, 한글 자음에 알맞은 폴더(예: '김철수' 입력 시 'ㄱ' 폴더 아래 '김철수' 폴더)를 생성하고 상담일지.json 구조를 자동 할당합니다.

🩺 ISAP 진료차트 상담 기록

의사들의 진료 처방 기법인 SOAP 형식을 변형하여 설계사 업무에 특화된 차트 작성 양식을 제공합니다.

구분 명칭 기록 요령
I (Inquiry) 고객 니즈 고객 가입 목적, 가족력, 질병 기왕력, 희망 월 보험료 규모 등 기록.
A (Analysis) 보장 분석 기존 보유하고 있는 담보 분석 결과 (진단비 과소, 만기 부족 여부 등) 기록.
P (Proposal) 맞춤 제안 부족한 담보를 메우기 위해 설계한 신규 청약 추천 및 당사 제안 내용 기록.
P (Post-Plan) 향후 계획 설계서 전달 약속일, 2차 내방 상담 조율 시간, 청약 체결 목표일 기록.

📍 내 주변 고객 지도 (터치 영업용)

  • 상단 [고객 위치 지도] 탭을 누르면 설계사 본인의 실시간 GPS 위치가 파란색 동그라미로 지도에 매핑됩니다.
  • 주소가 등록되어 있는 고객들의 위치 정보가 지도 위에 마커 핀으로 렌더링됩니다.
  • 마커를 클릭하면 고객의 요약 프로필과 함께 [상담 차트 열기] 링크가 제공되어 즉석에서 이력을 볼 수 있습니다.

📅 구글 캘린더 기념일 연동

  • 고객의 상세 페이지 상단 프로필 패널에 표시된 생년월일과 보험 계약일 옆의 **[+ (캘린더 추가)]** 아이콘을 클릭합니다.
  • 구글 캘린더 API를 호출하여 매년 잊지 않고 챙길 수 있는 반복 알림 일정(🎂 [고객명] 생일, 📜 [고객명] 계약일)이 캘린더에 연간 Recurring 규칙으로 즉시 주입됩니다.

💳 실시간 3D 디지털 명함 발급

  • 상단 [디지털 명함] 탭에서 본인의 소속, 이름, 전화번호, 이메일, 상담 카카오톡 주소를 설정합니다.
  • [명함 링크 복사] 버튼을 누르면 고객에게 보낼 수 있는 인터넷 링크가 자동 생성됩니다.
  • 고객이 스마트폰으로 이 링크를 받으면, 모바일 화면 전체에 설계사의 화려한 입체 명함(플립 회전 가능)과 카카오톡 직통 문의 버튼이 나타납니다.

5. 자주 묻는 질문 (FAQ)

Q. 모바일 폰으로 onions-lab.xyz에 들어갔을 때 '로컬 서버 오류'가 계속 뜹니다.
A. 모바일 기기는 PC와 같은 로컬 하드웨어가 없습니다. 화면 우측 상단의 [로컬 PC] 대신 [구글 클라우드] 모드를 눌러 전환하신 후 사용하시면 정상 로그인 및 스캔이 가능합니다.

Q. 구글 로그인 시 '안전하지 않은 앱' 혹은 '권한 없음' 경고창이 나옵니다.
A. 구글 콘솔(GCP)에서 OAuth 동의화면 제작 시 [게시(Publishing)]를 하지 않고 개발 테스트용 상태로 두어 나타나는 정상적인 절차입니다. 경고 하단의 [고급] -> [onions-lab.xyz(으)로 이동(안전하지 않음)]을 클릭하여 넘어가 주시면 정상 작동합니다.

Q. 고객 거주 주소를 넣었는데 지도에 마커가 생기지 않습니다.
A. 지오코딩 API 호출이 이루어지려면 주소가 명확해야 합니다. 동/호수 같은 상세 주소는 생략하고, 행정동(예: '서울특별시 강남구 테헤란로 152')까지만 주소록에 적어두시면 구글 지도 매핑 성공률이 크게 향상됩니다.
